# 短信平台客户端 Flutter 代码规范

## 1. 文件与目录
- 每个页面/组件/模型单独一个 Dart 文件，命名采用小写+下划线。
- 目录结构清晰，见 ARCHITECTURE.md。

## 2. 命名规范
- 类名、枚举名采用大驼峰（PascalCase）。
- 变量、方法、函数采用小驼峰（camelCase）。
- 常量全大写+下划线（MY_CONST）。

## 3. 代码风格
- 使用 2 空格缩进。
- 每行不超过 100 字符。
- 适当添加注释，复杂逻辑需有文档注释。
- UI 组件拆分，避免单文件过大。
- 遵循 Dart/Flutter 官方推荐的格式化（可用 `dart format` 工具）。

## 4. 国际化
- 所有界面文案、错误提示、API响应均用多语言资源，不直接写死在代码。
- l10n 资源文件命名如 `intl_zh.arb`、`intl_en.arb`。

## 5. 状态管理
- 推荐 Provider/Riverpod，避免全局变量。
- 状态变更通过 Provider/Consumer/Notifier 实现。

## 6. 网络请求
- 所有 API 请求统一封装，错误码和异常需有统一处理。
- 网络层与 UI 层分离。

## 7. 组件复用
- 通用组件（如按钮、卡片、弹窗）统一放在 widgets 目录。
- 组件参数命名清晰，支持自定义样式和行为。

## 8. 主题与样式
- 主题色、字体、圆角等统一在 theme.dart 配置。
- 支持浅色/深色模式。

## 9. 其它
- 遵循 SOLID 原则，保持代码可维护、可扩展。
- 定期代码审查，保持风格一致。
